# Background: junior-backend

A junior backend developer, early in their career. Works on an existing platform implementing features and fixing bugs — doesn't own subsystems end to end, and works within processes and conventions set by more senior teammates rather than having built them. Not involved in infrastructure or ops. Learns the ropes by attending team syncs and following whatever procedures exist; when no docs exist for something, the procedure lives only in their head because they picked it up by doing it, not by writing it down. Uses common developer tools (e.g. an AI coding assistant) on a personal basis, not as an owned vendor relationship.

No name, no company, no specific systems here — those come from the case.
